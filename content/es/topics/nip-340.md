NIP-340 propone convenciones para usar firma threshold [FROST](/es/topics/frost/) (Flexible Round-Optimized Schnorr Threshold) en Nostr. FROST permite que un grupo de firmantes controle colectivamente una identidad Nostr donde cualquier subconjunto t-de-n puede firmar eventos sin reconstruir la clave privada completa.

## Cómo funciona

Un quórum FROST distribuye key shares entre n participantes. Cualquier conjunto de t participantes (el threshold) puede colaborar para producir una firma Schnorr válida para la pubkey Nostr compartida. Ningún participante individual posee nunca la clave privada completa, y menos de t participantes no pueden firmar. El NIP define cómo coordinar rondas de firma, distribuir key shares y publicar eventos firmados mediante threshold a través de relays Nostr.

## Casos de uso

- **Cuentas organizacionales**: Un equipo de proyecto controla una identidad Nostr compartida sin que un solo miembro tenga autoridad total de firma
- **Custodia**: Firma multipartita para operaciones sensibles
- **Verificación de releases**: Firma descentralizada de eventos de release de software
