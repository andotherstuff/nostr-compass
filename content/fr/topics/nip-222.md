NIP-222 est une proposition de brouillon pour standardiser la manière dont les applications externes transmettent du contenu partagé à un client Nostr. L'objectif est un URI de partage basé sur un schéma qui peut préremplir le texte d'une note ou d'autre contenu sans dépendre de solutions de contournement via le presse-papiers.

## Ce que la proposition décrit

Le brouillon dans [PR #2266](https://github.com/nostr-protocol/nips/pull/2266) décrit un schéma d'URI pour les intentions de partage, permettant à un jeu, une application web ou une autre application externe d'invoquer un client Nostr avec des données structurées déjà attachées.

Cela compte parce que les flux de partage sur mobile et desktop sont fragmentés aujourd'hui. Certains clients s'appuient sur des liens d'application personnalisés, d'autres sur les feuilles de partage système, et certains se rabattent encore sur la copie de texte dans le presse-papiers en demandant à l'utilisateur de le coller manuellement.

## Pourquoi c'est important

Un URI standardisé d'intention de partage rendrait la publication inter-applications plus propre et plus prévisible. Si plusieurs clients Nostr supportent le même format de transfert, une application externe peut cibler la publication Nostr une seule fois au lieu de gérer chaque client individuellement.

Le brouillon est encore ouvert, donc les détails peuvent changer avant toute fusion. Le point important pour l'instant est la définition du problème : Nostr dispose d'un bon support URI pour ouvrir des entités existantes, mais il manque encore un chemin standard pour partager du nouveau contenu vers un client.

## Statut actuel

Ce n'est pas encore un NIP fusionné. Considérez cette page comme une documentation de la proposition active, et non comme un comportement protocolaire établi.
