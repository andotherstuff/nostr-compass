NIP-340 propõe convenções para usar assinaturas threshold [FROST](/pt/topics/frost/) (Flexible Round-Optimized Schnorr Threshold) no Nostr. FROST permite que um grupo de signers controle coletivamente uma identidade Nostr, em que quaisquer t de n membros podem assinar eventos sem reconstruir a chave privada completa.

## Como funciona

Um quórum FROST distribui shares de chave entre n participantes. Quaisquer t participantes, onde t é o threshold, podem colaborar para produzir uma assinatura Schnorr válida para a pubkey Nostr compartilhada. Nenhum participante individual jamais detém a chave privada completa, e menos de t participantes não conseguem assinar. O NIP define como coordenar rodadas de assinatura, distribuir shares de chave e publicar eventos assinados por threshold por meio de relays Nostr.

## Casos de uso

- **Contas organizacionais**: uma equipe de projeto controla uma identidade Nostr compartilhada sem que um único membro tenha autoridade total de assinatura
- **Custódia**: assinatura multipartes para operações sensíveis
- **Verificação de releases**: assinatura descentralizada de eventos de release de software
