NIP-222 é uma proposta de rascunho para padronizar como apps externos entregam conteúdo compartilhado para um cliente Nostr. O objetivo é um share intent baseado em URI que possa preencher previamente o texto de uma nota ou outro conteúdo sem depender de workarounds de clipboard.

## O Que Propõe

O rascunho no [PR #2266](https://github.com/nostr-protocol/nips/pull/2266) descreve um esquema URI para share intents para que um jogo, web app ou outra aplicação externa possa invocar um cliente Nostr com entrada estruturada já anexada.

Isso importa porque fluxos de compartilhamento mobile e desktop são fragmentados hoje. Alguns clientes dependem de links de app personalizados, alguns de share sheets do sistema, e alguns ainda recorrem a copiar texto para o clipboard e pedir ao usuário para colar manualmente.

## Por Que Importa

Um URI de share intent padronizado tornaria a publicação entre apps mais limpa e previsível. Se múltiplos clientes Nostr suportarem o mesmo formato de handoff, um app externo pode direcionar a publicação Nostr uma vez em vez de fazer tratamento especial para cada cliente que deseja suportar.

O rascunho ainda está aberto, então detalhes podem mudar antes de qualquer merge. O ponto importante por enquanto é a definição do problema: Nostr tem forte suporte a URI para abrir entidades existentes, mas ainda falta um caminho padrão para compartilhar conteúdo novo em um cliente.

## Status Atual

Este ainda não é um NIP mergeado. Trate esta página como documentação para a proposta ativa, não como comportamento de protocolo definido.
